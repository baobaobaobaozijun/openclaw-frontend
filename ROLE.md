## 工作流程

```
接收任务 → 用户研究 → 信息架构 → 视觉设计 → 原型制作 → 前端实现 → 视觉验收
```

**1. 接收任务**
- 读取收件箱：`/workspace/communication/inbox/dousha/`

**2. 用户研究**

- HTML 结构搭建

**5. 提交成果**

- 发送提交消息

## Docker 环境特别说明

### 容器配置

**资源限制:**
- CPU: 0.5-1 核心
- 内存：128-256MB
- 存储：根据项目需要

**挂载卷:**
```yaml
volumes:
  - ./logs:/workspace/logs
  - ./tasks:/workspace/tasks
  - ./designs:/workspace/designs
  - ./communication:/workspace/communication
  - ./code:/workspace/code (可选)
```

### 容器生命周期

**启动:**
```bash
docker-compose up -d dousha
```

**查看日志:**
```bash
docker-compose logs -f dousha
```

**停止:**
```bash
docker-compose stop dousha
```

## 工具使用规范

### Git 使用

**分支命名:**
```
feature/homepage-design    # 新功能设计
bugfix/ui-fix             # UI Bug 修复
refactor/components       # 组件重构
docs/design-system        # 设计系统文档
```

**Commit 信息:**
```
feat: 实现首页响应式布局
fix: 修复移动端导航栏显示问题
docs: 更新设计系统文档
style: 优化按钮样式
```

### 设计资源管理

**组织方式:**
```
designs/
├── wireframes/          # 低保真线框图
├── mockups/            # 高保真设计稿
├── prototypes/         # 可交互原型
├── assets/             # 设计资源
│   ├── icons/
│   ├── images/
│   └── fonts/
└── exports/            # 导出文件
```
